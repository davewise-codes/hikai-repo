## The Problem

**Title:** The bottleneck nobody talks about

**Body:**
Your product is evolving every day. Features ship. Bugs get fixed. The roadmap moves forward.

But outside of engineering, nobody knows.

And even when updates get shared, they're just a list — disconnected changes with no frame of reference. Nobody knows why they matter.

**Key insight:** The problem isn't writing — it's context. And no one outside your code has it.

---

## The Solution

**Title:** Automatic. Because you have better things to do.

**Body:**
Hikai doesn't just "help" with communication — it handles it.

Connect your repo. Hikai builds deep context about your product: what it does, how it's structured, why changes matter. Then it automatically narrates every meaningful update in language your audience understands.

No prompts. No editing. No bottleneck.

**Key points:**
- 100% automatic — no prompts, no drafts, no reviews.
- Context is the differentiator — Hikai writes with depth because it actually understands your product.
- Public by default — your timeline becomes a living, shareable page.

## A Timeline You Can Actually Use

**Title:** Not just updates. A navigable history of your product.

**Body:**

**Features:**

**Closing:** Context is everything. And Hikai has yours.

## Design Principles

### Brand Essence
- **Hikari** = luz, claridad, guía
- **Logo** = punto blanco en círculo negro (ser encontrado, destacar)
- **Sensación** = calma, precisión, confianza silenciosa
- **Mensaje de fondo** = "delega tu visibilidad, descansa"

### Visual Style
- Dark mode fijo (no theme switching)
- Whitespace generoso (más que v1)
- Tipografía como protagonista (compensar falta de assets)
- Sin decoración innecesaria
- Ritmo vertical: alternar densidad/respiro

### References
- **Intercom** (principal): estética, uso de imágenes atmosféricas
- **Sanity**: tipografía protagonista, jerarquía clara
- **Frontify**: uso tipográfico, premium por sustracción

---

## Section Backgrounds & Rhythm

Alternar fondos entre secciones para crear dinamismo visual y separación clara.

| Section | Background | Notes |

| Footer | bg-background | Clean |

### Overlay System (for image backgrounds)

Todas las imágenes llevan un sistema de overlays en capas:

```
Layer 1: Base gradient
         from-slate-950/80 via-slate-950/45 to-slate-950/70
         (asegura legibilidad del texto)

Layer 2: Grid pattern
         linear-gradient white/10
         bg-[size:88px_88px]
         opacity-25-30
         (añade textura sutil)

Layer 3: Color tint (opcional)
         from-slate-950/50 via-slate-900/30 to-slate-800/20
         (ajusta el tono general)
```

### Rhythm Principle

- Alternar: limpio → sutil → limpio → sutil
- Secciones con imágenes rompen el patrón (Hero, Visibility, Context)
- CTA final con color sólido cierra con contraste fuerte
- El ritmo debe sentirse natural, no mecánico

## Typography System

### Scale

| Element | Size (desktop) | Weight | Family | Notes |
|---------|----------------|--------|--------|-------|
| Hero title | 6xl-7xl | Black (900) | Sans | Máximo impacto |
| Section titles | 5xl-6xl | Black (900) | Sans | Protagonistas |
| Taglines/emphasis | 2xl-3xl | Regular (400) | Serif italic | Contraste elegante |
| Key insights | 2xl | Regular (400) | Serif italic | Bloques destacados |
| Body copy | lg | Regular (400) | Sans | Legibilidad |
| Secondary text | base-lg | Regular (400) | Sans, muted | Apoyo |
| Badges | xs | Semibold (600) | Sans, uppercase | Discretos |
| CTA buttons | base | Semibold (600) | Sans | Claros |

### Principles
- Headlines XXL: tamaños generosos, mucho más grandes que el típico SaaS
- Peso variable: Light para body, Black para títulos, contraste fuerte
- Serif solo para énfasis: taglines, insights, closings
- Una familia principal + serif para acentos
- El espacio negativo es parte del diseño tipográfico

## Design System Tokens

### Using @hikai/ui Token System

La landing DEBE usar el sistema de tokens de `@hikai/ui` definido en:
- `packages/ui/src/tokens/tokens.css` — Tokens estructurales
- `packages/tailwind-config/index.js` — Mapeo a Tailwind

### Tokens que usamos (ya disponibles)

| Category | Tokens | Usage in Landing |
|----------|--------|------------------|
| **Colors** | `background`, `foreground`, `muted`, `muted-foreground`, `primary`, `primary-foreground`, `border` | Todos los colores semánticos |
| **Border Radius** | `rounded-lg`, `rounded-xl`, `rounded-2xl`, `rounded-3xl` | Cards, containers, buttons |
| **Shadows** | `shadow-sm`, `shadow-md`, `shadow-lg`, `shadow-xl` | Cards, product mockups |
| **Z-Index** | `z-sticky`, `z-fixed` | Navigation |
| **Transitions** | `duration-normal`, `duration-slow`, `ease-out` | Hover states, scroll effects |
| **Font Families** | `font-sans`, `font-serif` | Headlines (sans), taglines (serif) |

### Tokens que necesitamos extender

**1. Typography Scale — Añadir tamaños XXL**

El sistema actual llega hasta `text-5xl` (48px). Necesitamos:

```js
// En packages/tailwind-config/index.js → theme.extend.fontSize
fontSize: {
  '6xl': ['3.75rem', { lineHeight: '1' }],    // 60px
  '7xl': ['4.5rem', { lineHeight: '1' }],     // 72px
  '8xl': ['6rem', { lineHeight: '1' }],       // 96px (opcional)
}
```

**2. Font Weight — Añadir font-black**

El sistema actual tiene hasta `font-bold` (700). Necesitamos:

```js
// En packages/tailwind-config/index.js → theme.extend.fontWeight
fontWeight: {
  black: '900',
}
```

**3. Custom Font Size Token (opcional)**

Para los tamaños específicos de la landing, podríamos añadir tokens semánticos:

```css
/* En packages/ui/src/tokens/tokens.css */
:root {
  --fontSize-hero: 4.5rem;      /* 72px - Hero title */
  --fontSize-section: 3rem;     /* 48px - Section titles */
  --fontSize-tagline: 1.875rem; /* 30px - Taglines */
}
```

### Tokens de Tailwind base que usamos (no requieren extensión)

- **Opacity**: `white/10`, `white/35`, `white/80`, etc. (Tailwind arbitrary values)
- **Backdrop**: `backdrop-blur`, `backdrop-blur-sm` (Tailwind plugin)
- **Spacing**: `py-24`, `gap-12`, `px-6` (Tailwind spacing scale)
- **Grid**: `grid-cols-3`, `grid-cols-[40%_60%]` (Tailwind grid)

### Color Tokens para Dark Mode Fijo

La landing usa dark mode fijo. Los valores de color vendrán del tema activo:

```css
/* Valores esperados (definidos en el tema) */
--background: /* slate-950 equivalent */
--foreground: /* white equivalent */
--muted: /* slate-800 equivalent */
--muted-foreground: /* slate-400 equivalent */
--primary: /* brand color */
--primary-foreground: /* white */
--border: /* slate-800 equivalent */
```

### Resumen de cambios requeridos en @hikai/ui

| File | Change | Priority |
|------|--------|----------|
| `packages/tailwind-config/index.js` | Añadir `fontSize: { '6xl', '7xl' }` | Required |
| `packages/tailwind-config/index.js` | Añadir `fontWeight: { black: '900' }` | Required |
| `packages/ui/src/tokens/tokens.css` | (Opcional) Añadir tokens semánticos de fontSize | Nice to have |

**IMPORTANTE**: Estos cambios se hacen en `packages/` para que estén disponibles en todo el monorepo, no solo en website.
